Gotcha: ConcurrentModificationException

Consider this code:

List<String> list = new ArrayList<>();
list.add("abc");
list.add("def");
list.add("ghi");    
list.add("jkl");
for(String s : list)
{
    list.add(s + s);
}
System.out.println(list);

What is it trying to do, and what does it actually do? Can you identify a way to fix it?

Warning

You should never catch ConcurrentModificationException, and doing so would not help you solve the problem here anyway.

For Reference: Gradle and CMake

AddressBook and ImageViewer (the Java and C++ versions anyway) are set up to use either the Gradle or CMake build tool. A build tool is used to manage compilation and related processes.

If you see any files with âgradleâ in the name, then youâre dealing with Gradle, typically used for Java projects (and Kotlin and other JVM-based languages).

To compile and run a Gradle-based project on Linux/MacOS, type the following in the main directory:

(Do the same on Windows too, but without â./â.)

You actually donât need Gradle installed yourself. The files gradlew and gradlew.bat are self-contained scripts that will automatically download Gradle and run it.
